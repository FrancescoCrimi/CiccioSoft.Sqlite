  # SqliteConnectionStringBuilder vs SqliteConnection Architecture\n\n## Executive Summary\n\n**Tua Proposta**: Spostare i default details da `SqliteConnectionStringBuilder` a `SqliteConnection`.\n\n**Mio Giudizio Architetturale**: **PARZIALMENTE CORRETTO** — con precisazioni importanti.\n\n**Status Attuale**: ⚠️ Anti-pattern, viola Separation of Concerns  \n**Proposta**: ✅ Corretta, ma va implementata strategicamente\n\n---\n\n## Problema Attuale: Violazione SoC (Separation of Concerns)\n\n### Cosa Fa Adesso SqliteConnectionStringBuilder\n\n```csharp\npublic class SqliteConnectionStringBuilder : DbConnectionStringBuilder\n{\n    /// Constructor con hard-coded defaults ← PROBLEMA\n    public SqliteConnectionStringBuilder()\n    {\n        DataSource = string.Empty;      // ← Dettaglio implementativo\n        Pooling = true;                 // ← Dettaglio implementativo\n        MaxPoolSize = 100;              // ← Dettaglio implementativo\n        DefaultTimeout = 30;            // ← Dettaglio implementativo\n    }\n}\n\n/// Registrazione dei defaults nello static constructor ← PROBLEMA\ninternal abstract class SqliteConnectionStringOption(string[] keys)\n{\n    static SqliteConnectionStringOption()\n    {\n        DefaultTimeout    = Register(..., 30, ...);           // Default = 30\n        Pooling           = Register(..., true);              // Default = true\n        MaxPoolSize       = Register(..., 100, ...);          // Default = 100\n        JournalMode       = Register(..., \"WAL\");             // Default = WAL\n    }\n}\n```\n\n### Due Livelli di Default (Code Smell!)\n\n```\n┌─────────────────────────────────────────────────┐\n│ SqliteConnectionStringBuilder.ctor()             │  LIVELLO 1\n│ ├─ Pooling = true                               │  Hard-coded in constructor\n│ ├─ MaxPoolSize = 100                            │\n│ └─ DefaultTimeout = 30                          │\n└─────────────────────────────────────────────────┘\n                    ↓\n┌─────────────────────────────────────────────────┐\n│ SqliteConnectionStringOption.static ctor()      │  LIVELLO 2\n│ ├─ Register(..., true)  ← Register(Pooling)    │  Duplicato!\n│ ├─ Register(..., 100)   ← Register(MaxPoolSize)│  Duplicato!\n│ └─ Register(..., 30)    ← Register(DefaultTimeout)\n└─────────────────────────────────────────────────┘\n```\n\n**Problema**: Defaults sono definiti in **2 posti diversi** — fonte di inconsistenza\n\n---\n\n## Analisi: Responsabilità Corrette\n\n### SqliteConnectionStringBuilder — Cosa DOVREBBE fare\n\n```\n✅ RESPONSABILITÀ CORRETTE:\n├─ Parsing della connection string\n├─ Esposizione property per ogni parametro\n├─ Validazione formale (ints negativi → clamp, enums validi)\n├─ Aliasing (e.g., \"DataSource\" → \"Data Source\")\n└─ Serializzazione back to string format\n\n❌ COSA NON DOVREBBE FARE:\n├─ Codificare i valori di default\n├─ Conoscere semantica di \"Pooling=true\" per SQLite\n├─ Conoscere che \"DefaultTimeout deve essere 30 secondi\"\n└─ Logica \"intelligent defaults\" (WAL, ForeignKeys ON, ecc)\n```\n\n### SqliteConnection — Cosa DOVREBBE fare\n\n```\n✅ RESPONSABILITÀ CORRETTE:\n├─ Applicare i defaults di SqliteConnectionStringBuilder\n├─ Interpretare i valori della connection string\n├─ Risolvere InMemoryMode() → quale PRAGMA usare\n├─ ApplyConnectionSettings() → SQL pragmas\n├─ Logica \"se non specificato ForeignKeys, allora ON\"\n└─ Comportamenti di alto livello della connessione\n\n✅ COSA FAREBBE ANCHE (Futura):\n└─ Applicare i defaults intelligenti (WAL, ForeignKeys ON)\n```\n\n---\n\n## Pattern Corretto: Settings Hierarchy\n\n### Prima (Attuale - Anti-pattern)\n\n```csharp\n// SBAGLIATO: Builder conosce defaults\npublic class SqliteConnectionStringBuilder\n{\n    public SqliteConnectionStringBuilder()\n    {\n        Pooling = true;          // ← Dettaglio implementativo\n        MaxPoolSize = 100;       // ← Dettaglio implementativo\n        DefaultTimeout = 30;     // ← Dettaglio implementativo\n    }\n}\n\n// SBAGLIATO: Connection reutilizza builder\npublic class SqliteConnection : DbConnection\n{\n    public override void Open()\n    {\n        var settings = new SqliteConnectionStringBuilder { ConnectionString = _connectionString };\n        var pooling = settings.Pooling;          // Prende da builder (che lo ha hard-coded)\n        var maxPoolSize = settings.MaxPoolSize;  // Prende da builder (che lo ha hard-coded)\n    }\n}\n```\n\n**Problema**: `SqliteConnectionStringBuilder` si trasforma da \"data holder\" a \"policy enforcer\"\n\n### Dopo (Proposta Corretta)\n\n```csharp\n// CORRETTO: Builder è SOLO un parser\npublic class SqliteConnectionStringBuilder : DbConnectionStringBuilder\n{\n    public SqliteConnectionStringBuilder()\n    {\n        // ❌ NO defaults qui!\n        // Solo esporre property per accesso ai valori\n    }\n    \n    // Solo property che delegano al SqliteConnectionStringOption registry\n    public int DefaultTimeout\n    {\n        get => SqliteConnectionStringOption.DefaultTimeout.GetValue(this);\n        set => SqliteConnectionStringOption.DefaultTimeout.SetValue(this, value);\n    }\n}\n\n// CORRETTO: Connection applica i defaults intelligenti\npublic class SqliteConnection : DbConnection\n{\n    // Costanti private che definiscono i defaults di QUESTA implementazione\n    private const int DEFAULT_TIMEOUT_SECONDS = 30;\n    private const int DEFAULT_MAX_POOL_SIZE = 100;\n    private const bool DEFAULT_POOLING = true;\n    private const string DEFAULT_JOURNAL_MODE = \"WAL\";\n\n    public override void Open()\n    {\n        lock (_syncRoot)\n        {\n            var settings = new SqliteConnectionStringBuilder \n            { \n                ConnectionString = _connectionString \n            };\n\n            // Applica i defaults di SqliteConnection\n            int timeout = ExtractValueOrDefault(\n                settings.DefaultTimeout,\n                DEFAULT_TIMEOUT_SECONDS);\n            \n            int maxPoolSize = ExtractValueOrDefault(\n                settings.MaxPoolSize,\n                DEFAULT_MAX_POOL_SIZE);\n\n            var session = SqliteConnectionPool.Rent(\n                _connectionString,\n                dataSource,\n                maxPoolSize,  // ← Default from SqliteConnection, not builder\n                openFlags);\n        }\n    }\n    \n    private int ExtractValueOrDefault(int? value, int defaultValue)\n        => value ?? defaultValue;\n}\n```\n\n---\n\n## Stratégia di Refactoring\n\n### Step 1: Rimuovere Defaults da SqliteConnectionStringBuilder\n\n```csharp\n// ❌ Rimuovi il constructor che impone defaults\npublic class SqliteConnectionStringBuilder : DbConnectionStringBuilder\n{\n    // ❌ DELETE:\n    // public SqliteConnectionStringBuilder()\n    // {\n    //     DataSource = string.Empty;\n    //     Pooling = true;\n    //     MaxPoolSize = 100;\n    //     DefaultTimeout = 30;\n    // }\n\n    // ✅ KEEP: Riassegnare il valore al ConnectionString parsing\n    public SqliteConnectionStringBuilder(string connectionString) \n        => ConnectionString = connectionString;\n}\n```\n\n### Step 2: Mantenere i Defaults in SqliteConnectionStringOption Registry\n\n```csharp\n// ✅ KEEP: I defaults rimangono qui per fornire \"fallback values\"\n// quando una proprietà non è esplicitamente specificata nella connection string\ninternal abstract class SqliteConnectionStringOption(string[] keys)\n{\n    static SqliteConnectionStringOption()\n    {\n        // Questi defaults sono SOLO per la gestione della connection string\n        // (e.g., se l'utente crea un builder senza passare nulla)\n        \n        DefaultTimeout    = Register(..., 30, ...);      // ← OK qui\n        Pooling           = Register(..., true);         // ← OK qui\n        MaxPoolSize       = Register(..., 100, ...);     // ← OK qui\n        JournalMode       = Register(..., \"WAL\");        // ← OK qui\n    }\n}\n```\n\n### Step 3: Centralizzare la Logica in SqliteConnection\n\n```csharp\npublic class SqliteConnection : DbConnection\n{\n    // Costanti che definiscono i comportamenti di DEFAULT di questa implementazione\n    private static class Defaults\n    {\n        public const int DefaultTimeout = 30;           // secondi\n        public const int MaxPoolSize = 100;             // connessioni\n        public const bool Pooling = true;               // abilitato\n        public const string JournalMode = \"WAL\";        // better concurrency\n        public const bool ForeignKeys = true;           // referential integrity\n    }\n\n    public override void Open()\n    {\n        lock (_syncRoot)\n        {\n            var settings = new SqliteConnectionStringBuilder \n            { \n                ConnectionString = _connectionString \n            };\n\n            // Applica i defaults intelligenti di SqliteConnection\n            int timeout = settings.DefaultTimeout ?? Defaults.DefaultTimeout;\n            int poolSize = settings.MaxPoolSize ?? Defaults.MaxPoolSize;\n            bool pooling = settings.Pooling ?? Defaults.Pooling;\n\n            // ... rest of implementation\n        }\n    }\n\n    private void ApplyConnectionSettings(Sqlite3 native)\n    {\n        // Intelligent defaults: ForeignKeys ON if not specified\n        bool foreignKeysSpecified = _settings.HasForeignKeys;\n        bool foreignKeysValue = foreignKeysSpecified \n            ? _settings.ForeignKeys ?? false \n            : Defaults.ForeignKeys;  // ← Default intelligente qui\n        \n        native.Execute($\"PRAGMA foreign_keys={(foreignKeysValue ? \"ON\" : \"OFF\")};\");\n\n        // Intelligent defaults: Journal Mode WAL if not specified\n        string journalMode = _settings.IsInMemoryMode()\n            ? \"DELETE\"  // WAL not supported in-memory\n            : (_settings.HasJournalMode && !string.IsNullOrWhiteSpace(_settings.JournalMode)\n                ? _settings.JournalMode\n                : Defaults.JournalMode);  // ← Default intelligente qui\n        \n        native.Execute($\"PRAGMA journal_mode={journalMode};\");\n    }\n}\n```\n\n---\n\n## Benefici dell'Architettura Corretta\n\n### 1. Single Source of Truth\n\n**Prima** (Attuale):\n```\nDefaults in 2 posti:\n├─ SqliteConnectionStringBuilder.ctor()\n└─ SqliteConnectionStringOption.Register()\n```\n\n**Dopo** (Proposta):\n```\nDefaults in 1 posto:\n└─ SqliteConnection.Defaults (static class)\n```\n\n### 2. Separation of Concerns\n\n| Classe | Responsabilità | Prima | Dopo |\n|--------|----------------|-------|------|\n| **SqliteConnectionStringBuilder** | Parse connection string | ❌ Implementa defaults | ✅ Solo parser |\n| **SqliteConnection** | Esegui connessione | ✅ Applica logica | ✅ Applica defaults + logica |\n| **SqliteConnectionStringOption** | Registry opzioni | ✅ Fallback values | ✅ Fallback values |\n\n### 3. Testabilità\n\n**Prima**:\n```csharp\n[Fact]\npublic void DefaultTimeout_defaults_to_30()\n{\n    var builder = new SqliteConnectionStringBuilder();\n    Assert.Equal(30, builder.DefaultTimeout);  // ← Testing builder behavior\n}\n```\n\n**Dopo**:\n```csharp\n[Fact]\npublic void Open_applies_default_timeout_30()\n{\n    var connection = new SqliteConnection();\n    connection.Open();\n    Assert.Equal(30, connection.DefaultTimeout);  // ← Testing connection behavior\n}\n```\n\n**Vantaggio**: Testi sono semanticamente corretti (si testano comportamenti, non configurazioni)\n\n### 4. Extensibilità\n\nFuturo: Sottoclass di `SqliteConnection` con defaults diversi\n\n```csharp\n// Sottoclass con defaults specifici per enterprise\npublic class EnterpriseSqliteConnection : SqliteConnection\n{\n    private new static class Defaults\n    {\n        public const int DefaultTimeout = 60;        // ← Timeout più lungo\n        public const int MaxPoolSize = 500;          // ← Pool più grande\n        public const string JournalMode = \"WAL2\";    // ← Journal mode avanzato (futuro)\n    }\n}\n\n// Vs.
